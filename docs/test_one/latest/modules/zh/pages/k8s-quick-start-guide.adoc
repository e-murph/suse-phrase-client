== 概述

当您的 SUSE Observability SaaS 实例完成设置和配置后，您将收到一封来自 SUSE Observability 的电子邮件，其中包含所需的登录详细信息。本快速入门指南将帮助您入门并将自己的数据导入 SUSE Observability SaaS 实例。

要将集群与 SUSE Observability 集成，您可以根据相应的环境选择以下指南之一。

=== 使用 Kubernetes 的先决条件

要设置 SUSE Observability Kubernetes 集成，您需要具备以下条件：

* 运行正常的 Kubernetes 集群。
* Helm 3.13.1 或更高版本。
* 拥有`create privileged pods`,`ClusterRoles` 和`ClusterRoleBindings` 权限的用户：
 ** 需要 ClusterRole 和 ClusterRoleBinding 来授予 SUSE Observability 代理访问 Kubernetes API 的权限。
 ** SUSE Observability 代理需要在特权 pod 中运行，才能收集网络连接信息和主机信息。

=== 设置 Kubernetes 集成

[CAUTION]
====
开始之前，请检查<<_prerequisites_for_kubernetes,Kubernetes 的先决条件>>。
====


要将 Kubernetes 集群中的数据导入 SUSE Observability，请按以下步骤操作：

. 将 SUSE Observability helm 资源库添加到本地 helm 客户端：
+
[,buildoutcfg]
----
 helm repo add suse-observability https://charts.rancher.com/server-charts/prime/suse-observability
 helm repo update
----

. 在 SUSE Observability UI 中，单击屏幕左上角打开主菜单，然后转到`StackPacks` >`Kubernetes` 。
. 安装 Kubernetes StackPack 的新实例：
 ** 指定*Kubernetes 群集名称*
  *** 该名称将用于在 SUSE Observability 中识别群集
 ** 点击*安装*。
. 安装 StackPack 后，使用 SUSE Observability UI 中提供的 helm 命令在集群上部署 SUSE Observability Agent、Cluster Agent、Checks Agent 和 kube-state-metrics。
 ** 一旦部署了代理，它们将开始收集数据并将其推送到 SUSE Observability

[CAUTION]
====
在自托管空气分接环境中运行时，首先按照xref:/k8s-suse-rancher-prime-agent-air-gapped.adoc[空气分接说明]准备代理安装。
====

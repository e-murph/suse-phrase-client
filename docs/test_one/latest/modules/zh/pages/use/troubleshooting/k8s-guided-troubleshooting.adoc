= 什么是引导式故障排除？
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE 可观察性

== 概述

使用 SUSE Observability 引导故障排除是一种强大的方法，可在整个故障排除过程中提供有针对性、可操作的见解，从而加快问题的解决。通过利用先进的算法，SUSE Observability 可根据您的特定环境提供故障排除提示、可视化帮助和逐步指导。这不仅简化了发现和解决问题的流程，还使站点可靠性工程师（SRE）能够更好地支持开发团队。

通过利用 SUSE Observability 的指导修复功能，工程师可以确保提供一致的高质量服务，并与其他团队成员分享他们的专业知识。此外，我们的修复指南可以轻松扩展或修改，以适应您的独特环境，使其成为维护服务可靠性和性能的宝贵工具。

预先配置的监控器可查看正确的内容，并在正确的时间发出警报，同时提供明确的提示，使工程师能够修复问题。该指南可帮助每位工程师立即了解需要采取哪些补救措施。此外，在问题得到解决之后，这些信息将支持无责事后调查的过程，以确定需要改进的地方。

== 通过指导故障排除来解决问题

为了快速补救，SUSE Observability 在一个屏幕中提供了清晰的问题报告和补救指南。它包含以下项目：

. 简要说明，以便向不太了解问题的人解释问题。
. 有关该问题的一些事实，如健康状况、触发时间和原因（如果存在）。
. 辅助指标表明问题是如何随时间演变的。
. 问题往往不是孤立发生的。有时，它们会导致其他问题，或者真正的问题是由不同的组件造成的。SUSE Observability 会跟踪所有组件之间的关联，并就相关问题向您发出警告。
. 修复指南本身可指导您逐步解决问题。

== 排除故障时使用固定项目

您可以将修复指南添加到*固定项目*中，以便在排除故障时随身携带。单击 "*添加到固定项目*"按钮，固定当前监视器的补救指南。现在，即使关闭触发的监视器，您也可以按照步骤进行操作。您可以从固定项目菜单访问所有固定的补救指南。完成故障排除后，只需从菜单中取消锁定指南即可。

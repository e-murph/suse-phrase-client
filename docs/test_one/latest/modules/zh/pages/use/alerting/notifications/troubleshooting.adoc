:revdate: 2025-07-10

:description: SUSE 可观察性

当通知通道无法发送通知时，通知概览将显示错误状态。检查错误：

. 点击通知
. 页面顶部的横幅显示了错误摘要
. 向下滚动到频道，点击受影响频道的 "错误 "链接，打开错误详细信息。

== 常见错误

=== 松弛

Slack 错误通常包含一个错误代码：

* `token_revoked`,`token_expired` 或`missing_scope` ：这些错误代码都表明 SUSE Observability 向 Slack 发布消息的授权存在问题。要解决这些问题，请重新创建通道，通过 Slack 重新授权 SUSE Observability，以便授予所有必要的权限。
* 有关其他错误代码，请参阅https://api.slack.com/automation/cli/errors[Slack API 错误代码]。

=== HTTP

大多数通道，特别是 webhook，都希望得到状态为 200 OK 的 HTTP 响应。其他答复被视为错误。HTTP 错误包含一条通常很短的信息和收到的状态代码，而不是 200 OK。

验证配置的 URL 和令牌是否正确，如果是 webhook，则验证是否正确处理了 POST 请求并返回了 200 OK 响应。

=== 其他

如果错误类型为 "其他"，SUSE Observability 很可能无法连接到外部服务或网络钩子。验证 URL 是否正确、可以解析并可被 SUSE Observability 访问。

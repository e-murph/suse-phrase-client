== 配置 Slack 通知

要向 Slack 发送通知，请按以下步骤操作：

. <<_connect_slack_workspace,将 SUSE Observability 连接到您的 Slack 工作区>>
. <<_select_a_slack_channel,选择要发送通知的 Slack 频道>>
. <<_add_and_test_the_channel,添加并测试通道>>

=== 连接 Slack 工作区

点击 "选择工作区 "按钮。这将打开一个 Slack 网页，SUSE Observability 会在该网页上请求允许列出频道和向 Slack 频道发送消息。确保在右上角选择了所需的 Slack 工作区。

点击 "允许 "继续。

=== 选择 Slack 频道

image::k8s/notifications-slack-channel-configuration.png[选择 Slack 频道]

回到 SUSE Observability，您现在可以从列表中选择 Slack 频道。选择需要发送通知信息的通道。

[NOTE]
====
私人频道不会自动列出。要选择私人频道，首先要邀请 SUSE Observability 机器人进入私人频道，方法是在频道中发送一条提及机器人的 Slack 消息`@SUSE Observability` ，然后选择 "邀请他们"。现在，频道将出现在列表中。
====


== 添加并测试通道

最后点击 "添加频道 "按钮。这将把频道添加到右侧的频道列表中。它会显示一个 "测试 "按钮。按下 以验证测试信息是否出现在所选的 Slack 频道中。

== 用于通知的 Slack 消息

打开通知时，会创建一条新的 Slack 消息。通常只有当健康状态发生变化时，才会更新此信息。在信息的线程中，每次更改也会显示为一条信息。最后，当通知关闭时，Slack 消息会再次更新（现在显示健康状态的条形图是灰色的），并在 Slack 线程中添加最后一条消息，说明关闭通知的原因。

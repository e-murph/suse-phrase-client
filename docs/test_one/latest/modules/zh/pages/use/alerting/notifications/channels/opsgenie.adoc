== 配置 Opsgenie 通知

要向 Opsgenie 发送通知，请按以下步骤操作：

. <<_create_integration,在 OpsGenie 中创建集成>>
. <<_create_and_test_the_channel,创建频道>>

=== 创建集成

为了将 SUSE Observability 与 OpsGenie 集成，需要创建一个全局 "API 密钥"。 每个通知的响应者可以不同，并将在 SUSE Observability 中配置。

在 Opsgenie 中，转到 "设置 \-> 集成"。

* 点击 "添加集成 "按钮
* 搜索 "API "集成类型。
* 为该集成选择一个名称，但_不要选择受让人团队_。

创建集成后，它将拥有一个 "API 密钥"。

=== 创建并测试通道

image::k8s/notifications-opsgenie-channel-configuration.png[创建频道]

为您的 OpsGenie 账户选择适当的区域，然后输入 "API 密钥"。 为通知选择响应者；_用户_、_团队_、_计划_和_升级_均可用于此目的。

点击 "添加频道 "按钮。这将把频道添加到右侧的频道列表中。它会显示一个 "测试 "按钮。按下它将在 OpsGenie 中生成测试警报。

== OpsGenie 警报通知

打开通知时，会创建一个新的 OpsGenie 警报。通常只有当健康状态发生变化时，才会更新此信息。关闭通知后，OpsGenie 警报会再次关闭。 警报上的注释包含关闭警报的原因。

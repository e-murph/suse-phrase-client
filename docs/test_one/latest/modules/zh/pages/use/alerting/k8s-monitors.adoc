== 概述

SUSE Observability 通过监控器对最佳实践进行了编码。SUSE Observability 监控不仅能监控指标、事件或日志，还能监控拓扑结构和元数据，以检测常见问题，确保符合和遵守行业标准。
SUSE Observability 将监控器与我们独有的故障排除智能相结合，可快速检测出相关问题，并就如何补救这些问题提出建议。这种积极主动的方法降低了未发现问题的风险，有助于维护健康稳健的 Kubernetes 环境。
监控器附有针对警报资源和监控条件的补救指南，以指导用户排除故障。

== 监测结果

监测结果显示在两个地方：

. 在所有资源的高亮页面上，可立即查看出现了什么问题以及如何补救。
. 在右侧面板上，如果你选择拓扑结构中的某个元素，就能快速显示该组件的问题。

== 补救指南

补救指南显示在监控器状态抽屉中，如果通过单击资源的 "突出显示 "视角上的信息来打开监控器，则会显示该抽屉。

= 痕迹视角
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE 可观察性

跟踪视角显示组件的跨度及其相关指标。
这样，您就可以直接在 SUSE Observability 中监控 IT 基础架构中应用程序的性能。

点击列表中的任意跨度，即可查看属于该跨度的后代跨度。
展开后，跨度的时间轴会显示每个子代开始和完成的时间。
同一跟踪中的父跨度默认为折叠，但也可根据需要展开。 +
根据 OpenTelemetry 服务名称的不同，跨度的颜色也不同。
在检查轨迹并查看其跨度列表时，可以单击任意跨度查看更多细节。

image::k8s/k8s-traces-perspective.png[痕迹视角]

== 过滤痕迹

通过跟踪筛选器，您可以根据跨度状态（错误、正常或未设置）、父类型（外部、内部或根）和持续时间细化所显示的跟踪。

除这些过滤器外，跟踪还与在 SUSE Observability UI 底部的*时间*线控件中选择的*时间窗口*相匹配。
调整时间窗口，只显示该时间段内的痕迹。

=== 筛选持续时间

如第二张截图所示，可以通过刷直方图为跨度 "持续时间 "选择持续时间间隔。
可以通过清除所有滤波器或刷新整个范围来实现放大。

== 跨度详情

在 SUSE Observability 中，xref:/use/views/k8s-view-structure.adoc[视图]会显示特定 Kubernetes 资源的 IT 基础设施子选区。
痕迹视角显示与资源相关的跨度及其后代。
由于后代跨距可以源于其他资源，因此可以从跨距详细信息导航到它。
父级跨度 "链接将扩展视图以显示完整的轨迹。

image::k8s/k8s-traces-perspective-span-details.png[后裔跨度详情]

上面两张图片通过展示一个结账服务来说明这些概念，该结账服务的主要职责是从购物车创建订单。
您可以看到下订单请求的跟踪及其跨度示例。
已突出显示一个子跨度，并显示其详细信息，包括与其相关的组件链接。

同样，刷描轨迹也会放大轨迹的特定时间段。
稍后单击可重置选择。

== 时间旅行

使用 "轨迹视角 "时，就像使用其他视角一样，你可以选择实时模式，也可以选择xref:/use/stackstate-ui/k8sTs-timeline-time-travel.adoc#_time_travel[时间旅行回到过去]。

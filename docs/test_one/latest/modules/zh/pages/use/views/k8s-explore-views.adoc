= 探索景观
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE 可观察性

== 概述

一般来说，SUSE Observability 中的xref:/use/views/k8s-view-structure.adoc[视图]概念允许您监控之前保存的 IT 景观区域。但很多时候，当你需要调查某个视图的子集时，你并不想失去该视图的范围。这就是*探索视图*发挥作用的地方。

为了保持特定视图（如`my view` ）的范围不变，所有应用于拓扑元素或元素选 项（如组件、关系、组）的调查操作都将自动在临时探索视图中打开，该视图位于您的起始视图 （如`my view / explore` ）之下。

在探索视图中自动打开的*调查操作*示例：

* 在xref:/use/views/k8s-topology-perspective.adoc[拓扑视角]中点击组件上的*+ 按钮*，可显示该组件的隐藏邻居
* 对一个组件执行了*快速操作*
* 在xref:/use/views/k8s-topology-perspective.adoc[拓扑视角]中*双击*拓扑元素（组件、关系、组），对其进行研究
* 点击拓扑元素（组件、关系、组）上的*"探索..."*链接或按钮，以对其进行研究

[TIP]
====
在现有探索视图中执行的调查操作*不会*打开一个新的探索视图，而是会更改所处探索视图的范围。
====


== 处理

自动创建探索视图时，也会定义其原始范围。如果您想在对探索视图进行某些更改后将其重置为原始范围，这将非常有用。

虽然探索视图只是临时性的，但如果以后想对其进行监控或审查，也可以将其保存为xref:/use/views/k8s-custom-views.adoc[自定义视图]。

如果不想保存探索视图，只需从该视图移开，或使用顶部导航栏上的面包屑回到开始时的视图。

image::k8s/k8s-explore-views.png[探索景观]

== 结构

探索视图与xref:/use/views/k8s-custom-views.adoc[自定义视图]具有相同的结构：它们具有相同的xref:/use/views/k8s-view-structure.adoc#_filters[筛选器]和xref:/use/views/k8s-view-structure.adoc#_perspectives[视角]。

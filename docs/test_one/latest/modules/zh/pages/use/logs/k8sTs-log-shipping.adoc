== 代理安装

=== Openshift

Openshift 不支持第三方日志转发器，但该平台有日志转发选项，可替代 SUSE Observability 舵图中的 promtail 配置。 有关如何配置的详细说明，请参阅正在运行的 SUSE Observability 实例上的 Kubernetes 堆栈包文档。

=== Kubernetes

SUSE Observability k8s Agent helm 图表默认配置通过随图表提供的 helm 值将日志运输设置为启用：

[,yaml]
----
logsAgent:
  # logsAgent.enabled -- Enable / disable k8s pod log collection
  enabled: true
----

上述操作将确保在每个节点上部署一个 promtail 容器，以收集日志并将其发送到 SUSE Observability。 对于不希望向 SUSE Observability 发送日志的部署，可将上述值设置为`false` 。

=== 运行额外的 Promtail 吊舱

SUSE Observability 使用经过调整的配置来摄取日志，这通常不符合辅助要求。 因此，无法为其他目标端点的日志摄取运行单独的配置，建议运行第二个 promtail pod，将这些要求作为代理舵手图部署的 promtail 的一个单独关注点来处理。

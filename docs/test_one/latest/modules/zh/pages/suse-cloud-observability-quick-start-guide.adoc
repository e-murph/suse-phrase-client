== 概述

从云提供商市场购买 SUSE Cloud Observability 后，您的 SUSE Cloud Observability 环境将被调配。

您将收到一封来自 SUSE Cloud Observability 的电子邮件，其中包含所需的登录详细信息和环境链接。本快速入门指南将帮助您入门并将自己的数据导入到 SUSE Cloud Observability 部署中。

== 入门 - 初始步骤

. 设置密码
. 登录您的 SUSE Cloud Observability 实例
. 为群集安装 SUSE Observability 代理
. 更改个人账户详细信息

=== 设置密码

来自 SUSE Cloud Observability 的电子邮件包含一个独特的链接，您可以通过该链接在账户上设置初始密码。 在登录和配置可观察性环境之前，必须执行此操作。

=== 访问 SUSE Observability 环境

单击您从 SUSE Cloud Observability 收到的电子邮件中的唯一链接，登录您的 SUSE Cloud Observability 环境。 输入密码后将进入配置屏幕，以便添加群集。

ifdef::feature-agent-install-cluster[]
=== 为群集安装 SUSE Observability 代理

SUSE Cloud Observability 使用 StackPacks，以便更轻松地配置下游集群并将数据导入可观察性环境。

起初，您会看到以下界面：在 SUSE Observability UI 中，单击屏幕左上角的 "打开主菜单"，然后转到`StackPacks` >`Kubernetes` 。

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

进入 Kubernetes StackPack 屏幕后，将集群添加到可观察性环境中就非常简单了。

输入要观察的群集名称，该名称不必与 "kubeconfig "中使用的群集名称一致。

Kubernetes 群集名称必须以小写字母或数字开头和结尾，并且只能由小写字母、数字、点和破折号（.）

输入名称并单击`Install` 按钮。 您将看到以下屏幕，群组标记为 "等待数据"。

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - 添加群集]

单击集群名称，这将扩展此部分并显示一系列信息，包括可用于现有集群的先决条件和命令，以便将它们添加到可观察性环境中。

查看信息，确保您对 Kubernetes 环境拥有正确的权限，确保您运行的是受支持的 Kubernetes 版本。

这些命令是您的可观察性部署所独有的，其中包括所需的 API 密钥和 URL。 为你的集群选择合适的命令，这里有针对 EKS、RKE、通用 Kubnernetes 集群等的部分。

这些命令将安装 SUSE Cloud Obervability 代理，并将群集连接到 SUSE Cloud Observability 环境。

您可以直接从用户界面使用这些命令，也可以按照xref:/k8s-quick-start-guide.adoc[快速入门指南]中 "部署 StackState 代理和群集代理 "部分提供的命令将其部署到下游群集上

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - 安装后]

连接群集后，SUSE Cloud Observability UI 中应出现绿色勾号。

此时，您就可以开始探索您的数据了。
endif::feature-agent-install-cluster[]

=== 更改个人账户详细信息

电子邮件中的第 4 步是更新您的个人信息。 点击电子邮件中的唯一链接，并根据需要添加个人基本信息。 如有需要，您还可以在此部分设置 2FA 身份验证。

=== 探索您的数据

要开始探索数据，点击屏幕左上角的主菜单，打开`Kubernetes` ，显示可观察项目列表。

image::accessing_views_1.png[SUSE Cloud Observability - 探索您的数据]

从基础架构部分选择 "群集"，该部分将显示受监控群集的列表，选择你的群集，就能看到许多内置视图中的一个。

image::accessing_views_2.png[SUSE 云计算可观察性 - 群集视图]

此时，您可以开始探索群集的数据，或添加更多群集来收集数据。

有关如何使用 SUSE Cloud Observability（包括创建自定义视图）的详细信息，请参阅标准文档。

=== SUSE 云计算可观察性限制

请注意，SUSE Cloud Observability 不提供开箱即用的 RED 信号（速率、错误和持续时间），只显示速率信号。 SUSE Rancher Prime 提供此功能。
如果客户需要这些信号来获得完整的 Observability 解决方案，请联系 SUSE 讨论 SUSE Rancher Prime，或者使用 OpenTelemetry 收集器收集这些数据。

== 其他信息

有关更多信息，包括前提条件和平台支持的 Kubernetes 版本，请参阅xref:/k8s-quick-start-guide.adoc[快速入门指南]。
